# ft-listing-compliance-seo

Catch risky claims early while preserving discoverability.

## Workflow

1. Parse all listing text blocks and image-overlay copy.
2. Run compliance screen using `references/risk-claim-checklist.md`.
3. Tag each claim:
   - Safe
   - Needs evidence
   - High-risk / rewrite required
4. Run SEO checks:
   - Primary keyword coverage
   - Attribute keywords (material/use-case/audience)
   - Redundancy and stuffing risks
5. Produce final rewrite suggestions with minimal meaning drift.

## Deliverables

- Risky claim table
- Rewritten safe alternatives
- Keyword coverage report
- Final publish checklist

## 中文版本

在保证可搜索性的前提下，提前拦截高风险文案。

### 工作流

1. 解析标题、卖点、详情、图片叠字等全部内容。
2. 依据 `references/risk-claim-checklist.md` 做合规筛查。
3. 风险分级：
   - Safe（安全）
   - Needs evidence（需证据支撑）
   - High-risk（高风险需重写）
4. 做 SEO 体检：
   - 主关键词覆盖
   - 属性关键词覆盖（材质/场景/人群）
   - 堆砌和重复风险
5. 产出低偏差改写方案，尽量保留原卖点。

### 质量标准

- 不写绝对化承诺，不写无证据认证。
- 改写后语义应与原文保持一致，不引入新事实。
